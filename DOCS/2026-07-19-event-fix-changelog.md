## Fix #1: PGRST204 Schema Cache & Save Event Error

**Symptom:** `400 (Bad Request) ... Could not find the 'temple_inclusions' column of 'events' in the schema cache`

**Root Cause:** Migration `005_hybrid_materials_operations` was never applied to the remote Supabase project (`uqbrewpejdnugxkywasc`). The local TypeScript types included `temple_inclusions`, but the remote database did not have the column.

**Resolution:**
- Applied migration `005_hybrid_materials_operations` to add `temple_inclusions TEXT` column to `public.events`
- Also made `event_materials.event_id` and `contributions.event_id` nullable (standalone operations support)
- Ran `NOTIFY pgrst, 'reload schema'` to hard-refresh PostgREST cache
- Verified via `information_schema.columns` — column confirmed as `text`

## Fix #2: Storage Bucket Poster Upload 400 Error

**Symptom:** `POST /storage/v1/object/event-posters/posters/... 400 (Bad Request)`

**Root Cause:** The `event-posters` and `payment-proofs` storage buckets did not exist in the project. `storage.buckets` was empty.

**Resolution:**
- Created `event-posters` bucket — public, 10 MB limit, images only (`image/jpeg`, `image/png`, `image/webp`, `image/gif`)
- Created `payment-proofs` bucket — private, 10 MB limit, images + PDF
- Applied storage RLS policies:
  - `event-posters`: authenticated users can INSERT/UPDATE/DELETE (admin-only)
  - `payment-proofs`: authenticated users can INSERT; SELECT limited to authenticated

**Code changes in `eventAdminService.ts`:**
- Removed `posters/` subfolder prefix from upload path (bucket root upload only)
- Added `sanitizeFileName()` helper — strips special characters, replaces spaces with `_`, caps base name at 80 chars
- Upload format: `{timestamp}_{sanitized_base}.{ext}`
